# AI 剧本创作提示词（专业版）

请将以下提示词完整发送给 AI（如 ChatGPT o1 / Claude 3.5 Sonnet / DeepSeek R1），它将为你生成一份高质量、无重复、可直接导入的游戏配置表。

---

## 复制以下内容发送给 AI：

```markdown
你现在是一位资深的互动小说游戏设计师。请根据以下**严格的格式规范**和**设计要求**，为我创作一份完整的《人生模拟器》游戏配置表。

## 一、核心设计要求 (Game Design Principles)

1.  **拒绝重复，强调渐进性**：
    *   **严禁**使用通用的填充事件（如反复出现的“你发呆了一天”）。
    *   事件必须与**年龄**强相关。0-5岁是启蒙，6-18岁是求学/成长，19-30岁是冒险/奋斗，31-60岁是权谋/积累，61+是传承/结局。
    *   随着年龄增长，事件的复杂度、风险和收益都应显著提升。

2.  **策略性与数值成长**：
    *   玩家的属性分配（智商、体魄、金钱、恶名等）必须真实影响游戏进程。
    *   **关键节点**必须包含【选择分支】，且不仅是简单的成功/失败，而是通过不同选项引导向不同的属性流派（例如：选择“武力夺取”增加恶名和金钱，选择“智取”增加智商和人脉）。
    *   结局的达成必须依赖于玩家长期的属性积累（例如：“魔道至尊”结局要求极高的【魔念】和【修为】）。

3.  **剧情连贯性**：
    *   设计至少 3 条完整的**事件链**。前一个事件的选择（通过 `后续事件ID`）应触发后一个事件，形成一个小故事（例如：捡到神秘蛋 -> 孵化/吃掉 -> 孵化出神兽/拉肚子）。

## 二、配置表格式规范 (Strict Format Rules)

请严格遵守以下格式，不要添加 Markdown 表格，直接输出纯文本。保持缩进（2个空格）。

### 1. 外观设置 (Theme)
```
游戏标题: [你的游戏名]
副标题: [副标题]
背景色: #121212
文字色: #e0e0e0
主题色: #bb86fc
辅助色: #03dac6
面板背景色: #1e1e1e
边框色: #333
属性点数: 20
```

### 2. 世界 (Worlds)
*   **要求**：至少设计 1 个完整世界，包含 4-6 个独特属性。
*   **格式**：
    `世界: 世界ID | 世界名称 | 描述 | 属性1,属性2,属性3...`
    *(例: `世界: w_fantasy | 修仙界 | 强者为尊 | 修为,魔念,寿元,根骨`)*

### 3. 天赋 (Talents)
*   **要求**：至少 8 个天赋，涵盖不同品质（0-4）。
*   **格式**：
    ```
    天赋: ID | 世界ID | 名称 | 描述 | 属性:值,属性:值...
      品质: 0-4
      文本: 抽取时的详细文本
    ```

### 4. 事件 (Events) —— 重点内容！
*   **要求**：至少 **40** 个事件，覆盖 0-100 岁。包含【条件触发】、【选择分支】和【连锁剧情】。
*   **注意**：所有符号请尽量使用**半角符号**（英文冒号`:`，英文逗号`,`）。虽然编辑器已做兼容，但标准格式更安全。
*   **格式**：
    ```
    事件: ID | 世界ID | MinAge | MaxAge | 事件内容文本 | 属性:值...
      once: true          (可选)
      概率: 0-100         (可选)
      条件: 属性 > 值      (可选，支持 > < = >= <= != has)
      选项: 文本 | 属性:值... | 后续事件ID (可选)
    ```
    *(注意：`选项` 行可以有多行，每行代表一个按钮。)*
    *(示例：`选项: 勇敢冲锋 | 体魄:2,恶名:1 | e_battle_win`)*

### 5. 结局 (Endings)
*   **要求**：至少 5 个结局（完美、优秀、普通、悲惨、隐藏）。
*   **重点**：必须使用 `条件` 字段来通过数值判定结局。
*   **格式**：
    ```
    结局: ID | 世界ID | 名称 | 简述 | 优先级(数字)
      条件: 属性 >= 值     (支持 > < = >= <= !=)
      内容: <p>结局详细描述（支持HTML标签）</p>
